# Plan — Topic Tags

**PRD:** [`1.5-topic-tags.prd.md`](../prds/1.5-topic-tags.prd.md) · TDD throughout (≥80% new code). **GATE 1 approved** (keyword-only) → build → `code-reviewer` + `database-reviewer` → **GATE 2**.

## Build order (each step independently runnable/testable)

1. **Migration + types** — `items.topics text[] default '{}'` + GIN index on `topics`. Patch `supabase/types.ts`. Apply to local + cloud (REST/migration). *Verify:* column exists, existing rows default `{}`.

2. **Slug normalizer (pure)** — `lib/topics/slug.ts`: `toSlug(raw)` → `[a-z0-9-]`, collapse, dedupe; `prettifyLabel(slug)`. Unit-tested.

3. **Keyword matcher (pure)** — `lib/topics/keywords.ts`: curated map of known model/lab/product patterns → canonical slug (seed ~20: qwen/qwen3, llama, gpt, claude, gemini, deepseek, mistral, grok, etc.). `extractTopics(title, summary): string[]` with **word-boundary** matching to avoid false positives; returns canonical slugs via `toSlug`. Unit-tested, no I/O.

3b. **Backfill script (optional)** — one-off to tag existing rows from the same matcher, so current 800+ items get topics without re-ingest.

4. **Tagging on ingest** — extend `persistItems`/ingest run: `topics = extractTopics(title, summary)` stored on insert. *Verify:* live ingest populates topics.

5. **Topic query** — `lib/feed/queries.ts`: `getItemsByTopic(slug, limit)` (`.contains('topics',[slug])`, newest first) + `getTrendingTopics(n)` (unnest + count + recency). Unit-tested against a mock.

6. **Topic chips** — `ItemCard` renders up to 3 topic chips → `/topic/[slug]`. Reuse finite-first styling.

7. **Topic page** — `app/topic/[slug]/page.tsx`: title = prettified slug, cross-source list via `getItemsByTopic`, finite-first + Show more (reuse `FeedList` with a topic prop, or a thin variant).

8. **Trending strip** — small header strip on `/` from `getTrendingTopics`, linking through. Keep finite (top ~8).

9. **Review + verify** — full vitest + tsc + lint; `code-reviewer` + `database-reviewer` (migration); live-verify a known model aggregates ≥2 sources across categories.

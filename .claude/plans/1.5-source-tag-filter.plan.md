# 1.5 — Feed filters: by source & by tag (PRD + plan)

**Slice tier:** standard (2–5 files) · **State model:** URL-based, shareable (chosen at preference-checkpoint)
**Origin:** `.claude/prds/backlog/app-feedback-v3.md` (items 11, 18, 21–25) and v2 (overwhelm / "click Reddit → only Reddit").

## Why (the ask)

From v3: *"I want to be able to select a tag… If I want to select Reddit, I want to click on it, and only Reddit pages can be seen. If I want to read arXiv papers, I should be able to select that tag."* The feed is **overwhelming**; the reader wants to slice it to one source or one topic.

## What already exists (do NOT rebuild)

- **Time range** — `FeedWindow` (Today/Week/Month/All), default Month, already filters on `published_at`. ✅
- **Category nav** — `FEED_SECTIONS` header tabs (All/Papers/Repos/Models/Companies/Social/Products). ✅
- **Sort** — Relevant/Newest/Top-starred. ✅
- **URL-state** — `?section=&sort=&window=&show=` via `searchParams`, preserved by `feedHref`. ✅
- **Source badge** — `ItemCard` already renders `source.name`. ✅

## The delta this slice adds

1. **Filter by source** — `?source=<source_id>`. Click an item's **source badge** → feed shows only that source. Active-filter pill with a one-tap clear.
2. **Filter by tag** — `?tag=<tag>`. Render `items.tags[]` as small chips on the card; click a tag → feed shows only items carrying it. Active-filter pill + clear.
3. Both **combine** with the existing category/window/sort (e.g. Reddit source + Week). Empty state when a combination yields nothing.

Out of scope (deliberate follow-ups): platform-grouping (one "Reddit" chip spanning multiple subreddit sources), a persisted Settings tab, top-N-per-source caps, keyword include/exclude, metric sliders, LLM digest. These are separate v3 slices.

## Design

```
searchParams ?source=<id>&tag=<t>  ──►  page.tsx (parse, validate)
        │                                   │
        │ preserved by feedHref()           ▼
        ▼                          FeedList (source, tag props)
  ActiveFilters pill (× clear)             │
        │                                   ▼
        └────────────────►  getFeedItems({ source, tag, ... })
                                    │  .eq("source_id", source)
                                    │  .contains("tags", [tag])
                                    ▼
                              ranked items
ItemCard: source badge → href(source=id) ; tag chip → href(tag=t)
```

- **URL is the single source of truth** (ECC web `URL-as-state`); no client store, shareable/bookmarkable, back-button works.
- Source filter uses **`source_id`** (stable, exact). The visible pill resolves the id → source name from the loaded rows (or a light lookup) so the URL stays exact while the UI stays human.
- Tag filter uses Postgres array containment (`.contains("tags", [tag])`).
- Validate/normalize both params at the page boundary (unknown/empty → ignored). Tag is untrusted text → treat as a value, never interpolate; encode in hrefs.

## TDD task order (RED → GREEN → REFACTOR)

1. **queries**: test `getFeedItems` applies source filter → implement `.eq`. Then tag filter → `.contains`. Then combined + absent-ignored.
2. **filterHref helper**: test add/drop/preserve of `source`,`tag`,`section`,`sort`,`window` (omit defaults) → implement.
3. **page.tsx**: wire params, validation, `feedHref` via helper, Suspense key includes source+tag.
4. **ItemCard**: source badge link + tag chips (component renders; visual-regression-friendly, minimal markup assertions per ECC web testing).
5. **ActiveFilters**: renders pills only when active; each "×" drops exactly one param.
6. **FeedList**: forward props; empty state on no match.
7. Run `vitest run --coverage` (≥80% on touched `lib/`), `tsc --noEmit`, `eslint`. Manual mobile check at 320/375 (touch targets, no overflow).

## Acceptance criteria

- [ ] Clicking a card's source badge navigates to `?source=<id>` and the feed shows only that source.
- [ ] Clicking a tag chip navigates to `?tag=<t>` and the feed shows only items with that tag.
- [ ] Filters combine with category + window + sort; the URL is shareable and the back button restores prior state.
- [ ] An `ActiveFilters` pill shows each active filter with a working one-tap clear; a full reset returns to `/`.
- [ ] Empty combinations render a clear empty state, not a broken/blank feed.
- [ ] All new logic unit-tested; `vitest`, `tsc`, `eslint` green; touch targets ≥44px at 320/375.

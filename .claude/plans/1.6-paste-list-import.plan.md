# 1.6 ‚Äî Paste-list auto-extraction (slice 2)

**Subphase:** 1.6 (Source discovery & onboarding) ‚Äî second thin vertical.
**Preference-checkpoint:** user chose "Paste-list auto-extraction" as the next slice, 2026-07-04.
**Tier:** standard ‚Äî untrusted paste input (¬ß12.7), but **no new remote fetch** (we do not
retrieve the pasted URLs, only extract + store them; real validation still happens at
*promote* time via the existing `validateFeedUrl`). So no fresh SSRF surface beyond what
slice 1 already covers.

## Goal
Turn a single paste ‚Äî a "who/what to follow" article or a bare list of feed URLs ‚Äî into many
`source_candidates` in one shot, instead of adding them one at a time. Each extracted URL lands
in the same rating queue (state=suggested) and flows through the slice-1 rate ‚Üí promote path
unchanged.

## Scope decision (KISS / ¬ß12.7)
- **In:** extract `http(s)` URLs directly from the *pasted text blob*, dedupe, store as candidates.
- **Out (documented):** fetching a remote article URL server-side to scrape its links. That would
  add an SSRF fetch surface for no extra Phase-1 value ‚Äî the user can paste the article's text or
  the URLs themselves. Deferred with the other 2.4 discovery work.

## Reuse (already in repo)
- `lib/ingestion/sanitize.ts` ‚Üí `sanitizeUrl` (http/https only, blocks javascript:/data:/file:),
  `sanitizeText`.
- `lib/candidates/persist.ts` ‚Üí mirror `createCandidate`'s sanitize+insert discipline; reuse
  `SourceCandidate` / `NewCandidate` types.
- Route + envelope pattern from `app/api/candidates/route.ts`.
- UI pattern from `components/sources/AddCandidateForm.tsx`.

## New pure lib ‚Äî `lib/candidates/extract.ts` (no side effects, easy to unit-test)
- `extractCandidateUrls(text: string): string[]`
  - Regex-scan for `https?://‚Ä¶` tokens; run each through `sanitizeUrl` (drops unsafe schemes).
  - Normalize + dedupe **within the paste** (case-insensitive host, strip trailing punctuation).
  - Cap at `MAX_IMPORT_URLS` (50) so a giant paste can't bulk-insert unbounded rows.
  - Empty / no-URL input ‚Üí `[]`.

## Persist addition ‚Äî `lib/candidates/persist.ts`
- `addCandidates(inputs: NewCandidate[], client?): Promise<{ added: number; skipped: number }>`
  - Query existing `handle_or_url` from `source_candidates` (all states) **and** `url` from
    `sources`; build a dedupe set.
  - Filter inputs against it (skip already-known URLs ‚Äî no dupes, no re-proposing a live source).
  - Bulk-`insert` the survivors (sanitized, state=suggested).
  - Return counts so the UI can say "Added N, skipped M (already known)". Throws on DB error.

## Route ‚Äî `POST /api/candidates/import` (thin, Zod, envelope)
- Body `{ platform: string(1..40), text: string(1..20_000), whySuggested?: string(max 500) }`.
- `extractCandidateUrls(text)` ‚Üí if empty, 200 `{ added: 0, skipped: 0 }` (nothing to do).
- Map URLs ‚Üí `NewCandidate[]` (shared platform + optional why) ‚Üí `addCandidates` ‚Üí envelope
  `{ success, data: { added, skipped } }`.

## UI ‚Äî extend `/sources`
- `components/sources/ImportListForm.tsx` (client): platform `<select>` + `<textarea>` (paste the
  list/article) + optional why ‚Üí POST `/api/candidates/import` ‚Üí shows "Added N ¬∑ skipped M" ‚Üí
  `router.refresh()` so new rows appear in the queue. Visible error on failure. 44px touch targets.
- `app/sources/page.tsx`: render `ImportListForm` alongside the existing single `AddCandidateForm`
  (small heading each: "Add one" / "Paste a list").

## Acceptance
- Paste a list/article ‚Üí each new `http(s)` URL becomes a suggested candidate in the queue.
- URLs already in the queue or already an active source are skipped (reported as `skipped`).
- Unsafe-scheme tokens (javascript:, data:, file:) never become candidates.
- Oversized paste is capped at 50 URLs; empty/no-URL paste is a no-op, not an error.
- tsc + eslint clean; unit tests green. (No new fetch ‚Üí inline security note, no full agent pass.)

## Out of scope (later 1.6 / 2.4)
- Server-side fetch+scrape of a remote article URL to pull its links (SSRF surface ‚Äî deferred).
- Title extraction per URL (needs a fetch); auto-rating; citation/mention-trail discovery (2.4).

## Build order (each step: test first)

### 1. Tooling & deps (no app logic)
- Add deps: `@supabase/supabase-js`, `rss-parser`, `zod`. Dev: `vitest`, `@vitest/coverage-v8`, `@playwright/test`.
- `vitest.config.ts` (coverage thresholds 80% on `lib/**`), `playwright.config.ts` (mobile viewport project).
- `package.json` scripts: `test`, `test:watch`, `test:e2e`, `typecheck`.
- `.env.local`/`.env.example`: confirm `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`.

### 2. Supabase client lib — `web/lib/supabase/`
- `types.ts` — generated DB types (committed).
- `server.ts` — server-only client using **service-role** key (ingestion + reads in Phase 1). Guard: throws if env missing at startup (§21).
- *(No browser client needed this slice — feed is server-rendered.)*

### 3. Text sanitizer — `web/lib/ingestion/sanitize.ts`  *(TDD)*
- `sanitizeText(raw: string): string` — strip HTML tags, decode entities, collapse whitespace, trim. Unit tests cover entity/tag/whitespace cases.

### 4. arXiv connector — `web/lib/ingestion/rss/arxiv.ts`  *(TDD)*
- Implements `Connector` from `types.ts`. `fetch` the feed → `rss-parser` parse → map to `NormalizedItem[]`: `title`, `url` (link), `summary` (full sanitized abstract), `author`, `publishedAt` (ISO), `category` (= source category), cap **50**.
- Sanitize every field. Non-fatal issues → `warnings` (never throw on a bad item — §12.7).
- Tests run against a **saved fixture** (`web/lib/ingestion/rss/__fixtures__/arxiv-cs-lg.xml`) — no live network.

### 5. Persistence — `web/lib/ingestion/persist.ts`  *(TDD)*
- `persistItems(result: IngestionResult)` → upsert into `items` on conflict `url` (`ignoreDuplicates: true`); update `sources.last_fetched`.
- Test: re-running the same fixture inserts N first time, 0 the second (dedupe).

### 6. Ingest trigger route — `web/app/api/ingest/run/route.ts`  *(TDD)*
- `POST /api/ingest/run?source=<id|all>` → loads active `rss` source(s), runs connector, persists, returns `ApiResponse` (`{ added, skipped, warnings }`). Zod-validate the query.
- Server-only; not the cron (1.3). Guarded so it isn't trivially callable in prod later.

### 7. Feed API — `web/app/api/items/route.ts`  *(TDD)*
- `GET /api/items` → recent items, newest first (`published_at desc`), paginated (default 50). Returns `ApiResponse<Item[]>` with `meta`.

### 8. Feed UI — `web/components/feed/`
- `FeedList.tsx` (server component) — fetches via the items query layer, renders list.
- `ItemCard.tsx` — **Editorial list** card: category eyebrow, title, author · relative date, → affordance; external link `target="_blank" rel="noopener noreferrer"`. Design tokens only (no magic values); semantic HTML; thumb-friendly tap target (≥44px).
- `Abstract.tsx` (client) — collapsed to ~3 lines, "Show more/less" toggle; compositor-friendly (no layout-thrash animation). Renders abstract as **text** (no raw HTML).
- Wire into `web/app/page.tsx` (replace placeholder shell content with the feed).

### 9. Seed — `supabase/seed.sql`
- Extend to 4 active arXiv sources: `cs.AI` (exists), add `cs.LG`, `cs.CL`, `cs.CV`. Priority/tags per category.

### 10. E2E — `web/e2e/feed.spec.ts`  *(Playwright)*
- Seed + ingest (or seed fixtures), load `/` at 375px, assert ≥1 arXiv item renders, abstract expands, link host is `arxiv.org`.

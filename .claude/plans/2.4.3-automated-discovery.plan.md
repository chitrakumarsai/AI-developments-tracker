# PRD + Plan — 2.4.3 Automated Source Discovery (Slice 1: outbound-link trail → rating queue)

**Phase:** 2.4 · **Status:** GATE 1 — awaiting plan approval
**Tier:** LARGE + **mandatory `security-reviewer`** — this fetches many *untrusted external pages* (a real SSRF/DoS surface), is owner-gated, and writes candidates.

---

## Why
CLAUDE.md §5 + §2.4: the strongest cross-platform discovery signal is "the outlets your already-trusted sources keep referencing." Automate finding those and feed them through the existing human rating gate — so new sources are proposed, never auto-activated.

## Preferences (checkpoint 2026-07-09)
- **Signal:** outbound-link trail — fetch recent items' pages from ACTIVE sources, extract external links, tally recurring domains not already tracked → propose their feeds.
- **Scope:** analyzer → existing rating queue, **manual owner trigger**. No cron, no email this slice.
- **Surface:** in-app on `/sources` (reuse the rating queue + suggested-candidate count).

## Hard constraints (from research)
- **Link-first storage:** items store no bodies/outbound-links → pages must be **fetched transiently** (extracted, not stored — stays link-first).
- **OpenAI quota-blocked:** heuristic only, no LLM extraction.
- **No HTML-parser dep:** use a bounded dependency-free `href` extractor (no rendering → safe). `node-html-parser` is the noted alternative.

## Scope (Slice 1)
Owner clicks **"Run discovery"** on `/sources` → the analyzer fetches a bounded set of recent item pages from active sources, extracts external hosts, tallies **how many distinct sources** reference each host, proposes the top recurring untracked hosts as `source_candidates` (`state=suggested`) via `addCandidates`, and they appear in the existing rating queue for keep/skip/star. Result toast: `Proposed N new · M already known`.

### Out of scope → later
Scheduled/cron discovery + periodic digest; email; LLM entity extraction; feed-URL auto-guessing (we propose the host; the existing promote-time `validateFeedUrl` resolves the real feed); author/summary signals.

## Safety bounds (untrusted external fetch — §12.7)
- `unsafeUrlReason` runs on **every** URL before fetch (item pages *and* extracted links). Manual redirect handling reusing the validate.ts pattern (re-guard each hop).
- Caps: `MAX_ITEMS_PER_SOURCE` (e.g. 5 most-recent), `MAX_TOTAL_FETCHES` (e.g. 40), per-fetch timeout (`FETCH_TIMEOUT_MS`), **response-size cap** + **`content-type: text/html` only** (skip binaries), `MAX_LINKS_PER_PAGE` (avoid a hostile page emitting 10k links). Fetch failures are non-fatal (collected as warnings).
- Proposed candidate URLs are only *stored* (not fetched again here); promote-time `validateFeedUrl` re-guards + validates before anything goes live.

## API contract
`POST /api/discovery/run` → `{ success, data: { scannedSources, fetched, hostsFound, proposed, skipped, warnings[] } }`. 401/403 (owner gate), 429 (rate-limit), 500 (prod-redacted). Reuse `"candidates"` rate-limit bucket.

## Security review outcome (2026-07-09)
`security-reviewer` found 3 real issues in the fetch surface — **all fixed + regression-tested**:
- **CRITICAL** — `res.text()` buffered the full body before the size cap (OOM DoS) → now streamed via
  `body.getReader()` with an incremental byte cap (`readCappedHtml`).
- **HIGH** — SSRF guard was hostname-string-only (DNS-rebinding bypass) → added a per-hop DNS
  resolution guard (`realResolveGuard` + `ipIsPrivate`, IPv4+IPv6) that rejects any host resolving to
  a private/loopback/link-local IP, before every fetch.
- **MEDIUM** — content-type check failed open on a missing header → now fails closed.

**Residual (accepted, tracked):** DNS-rebinding **TOCTOU** is reduced but not fully closed — the
runtime re-resolves at connect time. Decision (user, 2026-07-09): **commit now** (the feature is
owner-only, manual, rate-limited, unmerged) and add **`request-filtering-agent`** (connection pinning
to the validated IP) as a **FOLLOW-UP before the discovery feature is exposed in production.**

## Acceptance
1. Owner "Run discovery" proposes recurring untracked hosts as `suggested` candidates that appear in the rating queue; already-tracked/queued hosts are skipped (deduped).
2. No private/loopback/metadata host is ever fetched; fetch caps + timeouts hold; a hostile/huge page can't hang or flood the run.
3. Non-owner/anon → 403/401; run is rate-limited; 500s redacted in prod.
4. New logic TDD'd ≥80%; `security-reviewer` no CRIT/HIGH.

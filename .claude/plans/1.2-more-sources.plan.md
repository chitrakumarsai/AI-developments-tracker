## Key insight
The existing `arxivConnector` (`web/lib/ingestion/rss/arxiv.ts`) is **already generic** — it maps standard RSS fields (title, link, contentSnippet, creator, isoDate), sanitizes, dedupes by URL, and carries the SSRF guard. arXiv feeds are plain RSS 2.0. So "generic RSS connector" is mostly a **rename + generalize**, not new logic. Minimal risk.

## Build order (test-first)

### 1. Generalize the connector → `web/lib/ingestion/rss/rss.ts`
- Rename `arxivConnector` → `rssConnector` (generic). Keep identical fetch/parse/sanitize/SSRF logic; drop the arXiv-only framing in comments. Keep `parseRssFeed(xml, source)` pure + fixture-testable.
- Keep `web/lib/ingestion/rss/arxiv.ts` as a thin re-export (`export { rssConnector as arxivConnector }`) OR delete and update imports — **decision: delete + update imports** (cleaner, one name).
- Move/rename tests: `rss.test.ts`; keep the arXiv fixture (proves no regression) and add a **company-blog fixture** (`__fixtures__/blog-sample.xml`) with `content:encoded` full HTML to prove snippet-only + sanitize.

### 2. Connector registry — `web/lib/ingestion/registry.ts`  *(TDD)*
- `getConnector(ingestionType: IngestionType): Connector` → `{ rss: rssConnector }`. Unknown/`api`/`scrape`/`manual` → returns null (caller records a warning; not yet implemented).
- Unit test: returns rssConnector for `'rss'`, null for the others.

### 3. Ingest route uses the registry — `web/app/api/ingest/run/route.ts`
- Replace the hardcoded `arxivConnector` call with `getConnector(source.ingestion_type)`; skip + warn on sources with no registered connector. Keep the CRON_SECRET gate and per-source resilience.
- No longer filters to `ingestion_type='rss'` in the query — loads all `active` sources, dispatches per type (non-RSS get a "no connector yet" warning).

### 4. Seed company/lab blogs — `supabase/seed.sql`
- Add `Companies & Labs` RSS sources. **Candidate starter set (validate each fetches before keeping):** Hugging Face blog (`https://huggingface.co/blog/feed.xml`), Google DeepMind, OpenAI news, Google AI/Research, Meta AI. Ship only the ones that return a valid feed; note the rest as backlog candidates.
- Priorities/tags per source; `ingestion_type='rss'`, `status='active'`.

### 5. Verify against real feeds (live)
- Apply seed to cloud DB, run `POST /api/ingest/run?source=all`, confirm: arXiv still ingests (no regression), blog items land tagged `Companies & Labs`, feed shows a mix, re-run dedupes to 0.

### 6. Feed UI
- No new component work required — `ItemCard` already renders the `category` eyebrow, so blog items surface with their category automatically. (Functional filtering/chips stay in 1.5.)
- E2E (`feed.spec.ts`): extend to assert ≥2 distinct categories render. *(Optional this slice; keep if cheap.)*

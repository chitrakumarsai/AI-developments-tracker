## API contracts (ApiResponse envelope)

- `POST /api/sources` — body `{ name, url, category, ingestionType, tags[] }` (Zod, mirror
  promote's bounds). `requireOwner` → `enforceRateLimit("candidates", user:<id>)` → `validateFeedUrl`
  (422 on fail, no write) → `createSource` → `{ success:true, data:{ id } }`. 400 bad JSON/payload,
  403 non-owner, 429 limited.
- `POST /api/sources/[id]/ingest` — `requireOwner` → rate-limit → `runIngestion(getServerClient(),
  { sourceId: id })` → `{ success:true, data: RunSummary }`. 403/429/500 as usual.

## Design (ECC web/design-quality — anti-template)
Match the existing editorial/Swiss `/sources` surface: reuse tokens, `font-display`, rule borders,
uppercase-tracked labels. Catalog rows = tabular, hierarchy via scale; `status` as a small semantic
pill (active/paused), `last_fetched` relative, count `tabular-nums`. Designed hover/focus/active on
Add + Ingest; clear empty/error/pending states. Mobile-first (touch ≥48px), no horizontal overflow.

## Non-goals / guardrails
- No new dependencies. No schema migration. Don't touch the candidate rating-queue flow.
- Don't call `/api/ingest/run` from the browser (CRON_SECRET) — use the new owner route.
- Keep `page.tsx`, each route, and each component well under the 800-line cap.

## Test plan
- Unit: `createSource` sanitizes+inserts; `listSourcesWithCounts` maps counts incl. zero-item source.

- Component: AddSourceForm shows a 422 reason; IngestButton renders `Pulled N · M new` and disables
  while pending.

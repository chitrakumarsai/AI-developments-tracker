# 1.6 ‚Äî Source onboarding: candidate ‚Üí rate ‚Üí promote (slice 1)

**Subphase:** 1.6 (Source discovery & onboarding) ‚Äî first thin vertical.
**Preference-checkpoint:** user chose "Manual candidate ‚Üí rate ‚Üí promote" (defer paste-list auto-extraction), 2026-07-04.
**Tier:** large ‚Äî ingests untrusted external content (feed validation fetches a user-supplied URL) ‚Üí **mandatory `security-reviewer` / SSRF care** (¬ß12.5, ¬ß12.7). TDD throughout.

## Goal
Exercise the whole ¬ß5 rating gate end-to-end for ONE source: add a candidate ‚Üí it lands in
`source_candidates` (state=suggested) ‚Üí rate it (skip/keep/‚≠ê) ‚Üí promote a rated candidate to
`sources` (status=active) only after its feed/URL validates. Auto-extraction from a pasted
list is a follow-up slice.

## Reuse (already in repo)
- `lib/ingestion/net.ts` ‚Üí `unsafeUrlReason(url)` (SSRF guard: http/https only, blocks
  private/loopback/link-local/IMDS), `FETCH_TIMEOUT_MS`, `USER_AGENT`.
- `lib/ingestion/rss/rss.ts` ‚Üí `parseRssFeed(xml, ref)` to confirm a feed actually parses.
- `lib/ingestion/sanitize.ts` ‚Üí `sanitizeText` / `sanitizeUrl` for untrusted candidate fields.
- Patterns to mirror: `lib/views/persist.ts` (server-only + injectable client) and
  `app/api/views/route.ts` (thin Zod route + `{success,data,error}` envelope).

## Data model
No migration ‚Äî `source_candidates (id, platform, handle_or_url, why_suggested, sample_items
jsonb, rating int, state suggested|promoted|rejected, created_at)` and `sources` both exist.

## Logic libs (server-only, injectable client, TDD)
- `lib/candidates/persist.ts`
  - `createCandidate({platform, handleOrUrl, whySuggested?})` ‚Üí sanitize + insert (state=suggested).
  - `listSuggested()` ‚Üí candidates in the rating queue (state=suggested), newest first.
  - `reviewCandidate(id, {action:'rate'|'skip', rating?})` ‚Üí rate sets `rating`; skip sets
    state=rejected. Throws on DB error.
  - `promoteCandidate(id, {name, category, ingestionType, tags})` ‚Üí inserts a `sources` row
    (status=active) from the candidate + sets candidate state=promoted. (Validation happens in
    the route BEFORE this, so this stays a pure DB op.)
- `lib/candidates/validate.ts`
  - `validateFeedUrl(url): Promise<{ok:boolean; reason?:string; sampleCount?:number}>` ‚Äî
    `unsafeUrlReason` ‚Üí `fetch` (timeout + UA) ‚Üí `parseRssFeed` ‚Üí ok when ‚â•1 item parses.
    Never throws; returns a reason string on any failure. (api/scrape types: validate URL
    safety + reachability only, skip RSS parse ‚Äî RSS is the default and the manual path.)

## Routes (thin, Zod, envelope)
- `POST /api/candidates` ‚Äî `{platform, handleOrUrl, whySuggested?}` ‚Üí createCandidate.
- `POST /api/candidates/review` ‚Äî `{id:uuid, action:'rate'|'skip', rating?:1..5}` ‚Üí reviewCandidate.
- `POST /api/candidates/promote` ‚Äî `{id:uuid, name, category, ingestionType, tags?}` ‚Üí
  `validateFeedUrl` first; on failure return 422 `{reason}`; on success `promoteCandidate`.

## UI ‚Äî new `/sources` route
- `app/sources/page.tsx` (server): `listSuggested()`; renders the add form + rating queue.
  Resilient (DB error ‚Üí notice). Linked from the main header nav.
- `components/sources/AddCandidateForm.tsx` (client): platform + URL/handle + why ‚Üí POST ‚Üí
  `router.refresh()`; visible error.
- `components/sources/CandidateCard.tsx` (client): shows candidate; skip / keep / ‚≠ê buttons
  (POST review); a Promote control (name + category + type + tags) that POSTs promote and
  surfaces the validation reason on 422. `router.refresh()` on success.

## Acceptance
- Add a candidate ‚Üí appears in the `/sources` rating queue.
- Skip ‚Üí it leaves the queue (rejected); rate/‚≠ê ‚Üí rating stored.
- Promote a rated candidate with a REACHABLE feed ‚Üí new `sources` row (active), candidate=promoted,
  and its items start appearing after the next ingest.
- Promote with an unreachable/unsafe/non-feed URL ‚Üí blocked with a clear reason (422), no source created.
- SSRF: private/loopback/non-http URLs rejected by `unsafeUrlReason` before any fetch.
- tsc + eslint clean; unit tests green; `security-reviewer` pass (no CRITICAL/HIGH).

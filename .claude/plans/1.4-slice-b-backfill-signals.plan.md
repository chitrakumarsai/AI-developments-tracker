# Plan ‚Äî 1.4 Slice B: 1-month backfill + capture popularity signals

Source: `.claude/plans/1.4-relevance-overhaul.plan.md` (Slice B). Ranking (Slice A)
is only as good as its data. B makes every source store a numeric popularity
`metric` and seeds ~30 days of history, so the ranker stops falling back to
recency for HN/Reddit.

## Preferences (preference-checkpoint, 2026-07-03)
- **Scope:** all four sources in one slice.
- **arXiv signal:** recency-only now (metric null); HF Daily-Papers upvote proxy later.
- **Backfill:** one-off scratchpad script, run once (not shipped).
- **Reddit/HN floor:** keep top-N per source/window.

## Research findings (current code state)
- **GitHub** (`api/github.ts`) ‚Äî ALREADY 30-day `pushed:>=` window + `sort=stars` +
  `metric: stars`. Done for B. "Capture forks too" needs a new DB column and doesn't
  fit the single `metric` field ‚Üí **proposed: skip forks this slice** (stars already
  drive popularity). Decide at GATE 1.
- **HN** (`api/hackernews.ts`) ‚Äî captures `points` but only into `summary` text, not
  `metric`. Needs `metric: points` + a 30-day `created_at_i>` filter in the query.
- **arXiv** ‚Äî generic RSS (`rss/rss.ts`), shallow feed, no metric. Month-deep +
  recency-only = thousands of papers ordered only by date (noise). **Proposed:** add a
  bounded arXiv API connector (date-range, hard per-category cap), metric stays null.
- **Reddit** ‚Äî NO connector exists; not seeded as `api`. Needs a new `reddit.com`
  connector via `top.json?t=month` (score ‚Üí metric, top-N per sub, link-first).
- Slice A `rankItems` already log-normalizes `metric` per source ‚Üí **no ranking changes**;
  HN/Reddit start ranking on popularity the moment they store `metric`.

## Tasks (ordered, TDD; connectors ingest untrusted input ‚Üí security-reviewer pass)

### B1 ‚Äî HN: store points as metric + 30-day window
- `parseHnSearch`: set `metric: points` (already computed for the gate).
- Add `created_at_i>SINCE` via `numericFilters` (points-filtering is rejected by
  Algolia; time filtering is allowed). Add a small URL builder like GitHub's, or bake
  the window into the connector. Keep `MIN_POINTS` + `MAX_ITEMS` as the top-N floor.
- Update fixture + tests. Smallest change; do first.

### B2 ‚Äî Reddit: new API connector (top-of-month, score‚Üímetric)
- New `api/reddit.ts` following the house pattern (pure `parseReddit` + I/O wrapper,
  sanitize every field, warns-not-throws, SSRF guard via `unsafeUrlReason`).
- Fetch `https://www.reddit.com/r/<sub>/top.json?t=month&limit=<N>`; link-first: prefer
  the post's external `url`, else the `permalink` discussion page. Store `score` ‚Üí metric.
- Top-N per subreddit (limit param). Register `www.reddit.com`/`reddit.com` in `api/router.ts`.
- Fixtures (top.json sample) + tests. Seed the existing subs (r/MachineLearning,
  r/LocalLLaMA) as `api` sources. (New subs from feedback = Slice C.)

### B3 ‚Äî arXiv: bounded API connector, date-range, metric null
- New `api/arxiv.ts` (or extend RSS): query `export.arxiv.org/api/query` with a
  `submittedDate:[SINCE TO NOW]` range and a hard `max_results` cap per category so a
  month doesn't flood the feed. metric omitted (recency-only per preference).
- Atom response ‚Üí reuse the RSS/Atom parse path where possible; sanitize; tests + fixture.
- Register `export.arxiv.org` in the router; migrate the 4 arXiv sources RSS‚Üíapi (or keep
  RSS and add API only for backfill). **GATE 1: bounded cap of ~50 most-recent/category.**

### B4 ‚Äî GitHub: add forks (GATE 1: approved)
- Migration: add `forks integer` to `items`; add `forks` to `ItemRow` + `NormalizedItem`.
- Connector: keep `metric = stars` (drives the 1.5 star badge, unchanged) and also store
  `forks`. Blend into ranking via an **effective popularity = metric + (forks ?? 0)**:
  add optional `forks` to `RankableItem` and use it in `rankItems`' map (additive ‚Äî forks
  undefined ‚Üí +0, so Slice A tests stay green; badge still reads stars).
- `persist.ts`: include `forks` in the upsert column set.

### B5 ‚Äî One-off backfill script (scratchpad)
- `scratchpad/backfill-30d.mjs`: load active sources from Supabase, run each connector
  with the wide window, upsert via existing `persist.ts` (dedupe on url). Run once to seed
  history; print a per-source count + warnings summary. Not committed to the app.

### B6 ‚Äî Verify
- Full test suite + typecheck; run the backfill; spot-check `/api/items?sort=relevant`
  now interleaves HN/Reddit by real score, not just recency. security-reviewer pass on the
  new connectors before GATE 2.

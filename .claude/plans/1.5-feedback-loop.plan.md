# 1.5 — Feedback loop (thumbs + read-state) — write path + filters + ranking

**Subphase:** 1.5 (Filters + UI/UX polish) · completes the deferred 1.4 feedback interaction.
**Preference-checkpoint:** user chose **"Full loop in one slice"** (write path + optimistic UI + ranking hook + `?state=` filter + ActiveFilters pill), 2026-07-04.
**Tier:** large — touches user data → **mandatory `security-reviewer` pass** (§12.5), TDD throughout.

## Decisions (why)
- **Denormalize current vote onto `items`.** Add nullable `items.feedback_value` (enum `feedback_value`). The `feedback` table stays an append-only history log; **all query/filter/rank reads use `items` columns only** — no fragile PostgREST embedded-table filters, and the existing fake test client already handles column predicates. One column, one migration.
- **Two thin routes over a testable lib.** Write logic lives in `lib/feedback/record.ts` (pure, injectable client) so routes are thin and unit-testable without HTTP.
- **New URL param `state`** (values `unread | liked | hide-down`). Cannot reuse `show` — it already means paging size in `filterHref`.
- **Service-role, server-only** writes (Phase 1, no auth). Zod-validated bodies; `itemId` must be a uuid; `value ∈ {up,down}` or null (clear).
- **Mark-read via `navigator.sendBeacon`** on opening the item (outbound link) — needs a small client wrapper.

## Data model
- Migration `supabase/migrations/20260704000001_items_feedback_value.sql`:
  `alter table items add column if not exists feedback_value feedback_value;`
  Apply to hosted DB via **native macOS psql** (IPv6; Docker psql can't route — known drift lesson).
- `ItemRow` gains `feedback_value: 'up' | 'down' | null`.

## Write path
- `lib/feedback/record.ts`
  - `recordVote({ itemId, value }, client)` → insert `feedback` history row (when value≠null) + `update items set feedback_value=value`. Returns void; throws on error.
  - `markRead(itemId, client)` → `update items set read_state=true`.
- `POST /api/feedback` — Zod `{ itemId: uuid, value: 'up'|'down'|null }` → `recordVote`. API envelope.
- `POST /api/items/read` — Zod `{ itemId: uuid }` → `markRead`. Envelope. (sendBeacon target.)

## Client UI
- `components/feed/FeedbackControls.tsx` (`"use client"`) — 👍/👎 buttons, optimistic local state, POST `/api/feedback`, **rollback + visible error on failure** (ECC react patterns). 44px targets. Initial vote from `item.feedback_value`.
- `components/feed/OpenAtSourceLink.tsx` (`"use client"`) — wraps the external anchor (title + "Read"); fires `sendBeacon('/api/items/read', {itemId})` on click, then normal navigation. `rel="noopener noreferrer"`.
- `ItemCard` stays a Server Component; renders both islands, passes `item.id` / `item.feedback_value`.

## Read path (filter + rank)
- `lib/feed/queries.ts` — `FeedQuery.state?: FeedState`. Apply:
  - `unread` → `.eq('read_state', false)`
  - `liked` → `.eq('feedback_value', 'up')`
  - `hide-down` → `.or('feedback_value.is.null,feedback_value.eq.up')`
- `lib/ranking/score.ts` — small multiplier: `down` demoted, `up` boosted (applies even when no `state` filter active). Reads `item.feedback_value` from the already-`*`-selected pool.
- `lib/feed/types.ts` — add `FeedState = 'unread' | 'liked' | 'hide-down'`.
- `lib/feed/filterHref.ts` — add `state?` to `FeedHrefParams`; emit `?state=`.

## Wiring
- `components/feed/FeedList.tsx` — thread `state` → context → `getFeedItems`; empty-state considers it.
- `components/feed/ActiveFilters.tsx` — clearable `state` pill.
- `app/page.tsx` — parse `state`, segmented control (All | Unread | Liked | Hide 👎), thread to FeedList + Suspense key + search hidden fields; carry `state` through nav/window/sort `feedHref` calls.

## TDD order

## Acceptance

- `?state=unread|liked|hide-down` narrows the feed and combines with category/source/tag/q/window/sort; pill clears it.
- Downvoted items sink, upvoted rise in the default Relevant order.
- tsc + eslint clean; all tests green; `security-reviewer` pass with no CRITICAL/HIGH.

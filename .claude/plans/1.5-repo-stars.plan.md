# Plan — Repo/Model popularity metric (stars & likes)

**Subphase:** 1.5 · **Type:** standard+ (migration + connector capture + UI + sort) → `database-reviewer` on migration. Pairs naturally with the topic-tags slice (same layers: new column, connector map, `ItemCard`, feed query) — build together in one fresh session.

## Problem (user feedback)
Under **Repos** there's no star count, and no way to sort by it. Same for Hugging Face pages — likes exist but only as prose in the summary, not shown as a rating or sortable.

## Findings (current code)
- `lib/ingestion/api/github.ts` fetches repos but **discards `stargazers_count`** — never mapped onto the item.
- `lib/ingestion/api/huggingface.ts` puts HF `likes` into the summary **text** only, not a structured field.
- `items` has no numeric popularity column.

## Scope (thin)
- Add `items.metric int` (nullable) — a generic popularity number (GitHub stars, HF likes). Keep it one column, not per-source, so the feed can sort uniformly. GIN/btree index on `metric` for sort.
- `NormalizedItem.metric?: number`; connectors set it: GitHub → `stargazers_count`, HF models → `likes` (HF papers → upvotes if present, else null).
- `persistItems` stores `metric`.
- `ItemCard`: when `metric != null`, show a compact badge (e.g. `★ 1,234` for repos, `♥ 1,234` for HF) near the date/eyebrow.
- Sort control: in **Repos** (and Models) allow `?sort=stars` → query orders by `metric desc nulls last`; default stays recency. Add a small toggle (Newest | Top-starred) reusing URL-as-state.

## Out of scope
- Historical star deltas/trend; per-source metric semantics beyond a single number; backfill beyond a one-off script.

## Build order (TDD)
1. Migration `items.metric int` + index; patch `supabase/types.ts`; apply local + cloud (REST). Backfill script optional.
2. `NormalizedItem.metric` in `types.ts`; github.ts maps `stargazers_count` (+fixture/test); huggingface.ts maps `likes` (+test).
3. `persistItems` writes `metric` (+test).
4. `getRecentItems`/`getItemsByTopic` accept `sort: 'recent' | 'metric'` (+test).
5. `ItemCard` badge; sort toggle in `page.tsx` for repos/models sections.
6. Review (`code-reviewer` + `database-reviewer`) + live-verify: repos sortable by stars, HF likes shown.

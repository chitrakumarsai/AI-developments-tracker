## Summary
Slice B makes every source store a numeric popularity signal and bounds each
connector to ~30 days, so the Slice A ranker stops falling back to recency for
HN/Reddit:
- **B1 HN** ‚Äî stores `points` as `metric`; injects a rolling 30-day
  `created_at_i>` window via Algolia `numericFilters`.
- **B2 Reddit** ‚Äî new `reddit.com`/`www.reddit.com` connector (`top.json?t=month`),
  `score`‚Üí`metric`, link-first (external url ‚Üí permalink fallback), top-N per sub.
- **B3 arXiv** ‚Äî new bounded `export.arxiv.org` API connector (submittedDate desc,
  hard 50/category cap), Atom parsing delegated to the shared `parseRssFeed`;
  metric stays null (recency-only per preference). Migrated the 4 arXiv sources
  RSS‚Üíapi in `seed.sql`.
- **B4 GitHub forks** ‚Äî new `forks integer` column (migration
  `20260703000001_items_forks.sql`); connector captures `forks_count`; ranker
  blends **effective popularity = metric + (forks ?? 0)** (badge/Top-starred keep
  reading `metric`=stars); `persist.ts` upserts + refreshes forks.
- **B5 backfill** ‚Äî one-off scratchpad script (`backfill-30d.mjs`, NOT shipped)
  that drives the existing `/api/ingest/run?source=all`.

## Security findings (security-reviewer, untrusted-ingestion pass)

### CRITICAL ‚Äî None.

### HIGH
- **H1 ‚Äî URL builders threw outside the connector try/catch** (warns-not-throws
  violation, ¬ß12.7). `buildHnSearchUrl`/`buildArxivQueryUrl`/`buildGithubSearchUrl`
  call `new URL(source.url)` before the connector's try block, so a malformed
  `source.url` could reject and abort a multi-source run (the `/api/ingest/run`
  loop has no per-connector catch). **RESOLVED**: each connector now guards the
  build in try/catch and returns an "invalid source URL" warning instead of
  throwing. Covered by new tests in `hackernews.test.ts` + `arxiv.test.ts`.

### MEDIUM (tracked follow-ups ‚Äî repo-wide, not Slice-B regressions)
- **M1 ‚Äî SSRF guard checks only the initial URL, not redirect targets.** `fetch`
  follows redirects; `unsafeUrlReason` validates only the requested URL. Slice B
  widens the trusted-host set (reddit.com, export.arxiv.org). Follow-up: harden
  `net.ts` (e.g. `redirect:"manual"` + re-validate `Location`, or check
  `response.url`). Applies to all connectors.
- **M2 ‚Äî No response-size cap before parsing.** `response.json()/text()` is
  unbounded; only item counts are capped post-parse. Follow-up: shared
  Content-Length / streaming cap in `net.ts`.

### LOW
- **L1 ‚Äî Reddit permalink built by string concatenation.** **RESOLVED**: now
  resolved via `new URL(permalink, REDDIT_ORIGIN)` (`permalinkUrl` helper).
- **L2 ‚Äî Untrusted values interpolated into warning strings** (author/objectID).
  Informational only (diagnostic strings, React-escaped if ever shown). Optional
  future log-hygiene pass via `sanitizeText`.

### Confirmed safe
- All new connectors sanitize every untrusted field (`sanitizeText`/`sanitizeUrl`)
  matching the house pattern; arXiv delegates to the already-sanitizing
  `parseRssFeed`. SSRF guard runs on the actually-fetched (built) URL. No secret
  exposure (`GITHUB_TOKEN` header-only, never logged). Item caps (50) + fetch
  timeouts present. `persist.ts` `forks` flows through parameterized supabase-js
  ‚Äî no injection.

## Follow-ups (not blocking this slice)
- M1 (redirect SSRF) + M2 (response-size cap) ‚Üí a `net.ts` hardening slice.
- L2 (sanitize warning strings) ‚Üí optional log hygiene.

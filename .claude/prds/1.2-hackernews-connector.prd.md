# PRD — Subphase 1.2 Slice 5: Hacker News connector

**Category:** Social / Discussion · **Type:** `api` (third provider under the host router) · Builds on GitHub + Hugging Face connectors.

## Problem
The catalog has papers, blogs, repos, and HF models, but no **discussion layer**. Direct X/LinkedIn ingestion is ToS-restricted; Hacker News is the ingestible, high-signal social surface for AI (via the keyless Algolia HN Search API).

## Goal
Add one HN provider that surfaces **AI-filtered, high-score stories**, link-first, with zero new architecture — it slots into the existing `api` host router next to GitHub/HF.

## Scope (this slice)
- Algolia HN Search API (`hn.algolia.com/api/v1/search`), query + `points>=` threshold encoded in `sources.url`.
- Link-first: external story `url`; text posts (Ask/Show HN, no url) fall back to `news.ycombinator.com/item?id={objectID}`.
- Summary = `{points} points · {num_comments} comments`.
- Keyless — no token.

## Out of scope
- Reddit, newsletters, X/LinkedIn aggregators — later 1.2 slices.
- Per-story comment ingestion; summaries/relevance (1.4).

## Preferences (locked)
| Decision | Choice |
|---|---|
| HN scope | AI-filtered high-score stories via Algolia Search API |
| Seed rows | Two — `query=AI` and `query=LLM` (dedupe handles overlap) |

## Acceptance criteria
- [ ] `hn.algolia.com` routes to the HN connector via the api router.
- [ ] Stories map to Social / Discussion; external link preferred, HN-thread fallback for text posts.
- [ ] All fields sanitized; SSRF guard on the fetch URL; warns-not-throws.
- [ ] Two `sources` rows added; new query = catalog edit, no code.
- [ ] Tests green, ≥80% new-code coverage; full suite + typecheck + lint clean.
- [ ] Live-verified; re-run dedupes to 0 on unique `url`.

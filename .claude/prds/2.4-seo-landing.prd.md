## Problem

The site (theaichronicles.ai) needs to be **discoverable** — findable via search, and
rich when a link is shared — **without exposing the full personalized app** to anonymous
visitors. Today `/` is a fully public read-only feed (2.1 model); there is no marketing
surface, no share cards, no sitemap/robots, and no gate between "what the world sees" and
"the app you sign in to use."

## Goal

Split a **public marketing surface** (indexable, shareable) from the **gated app**
(personalized experience), and give the public surface real SEO treatment.

## Decisions locked (preference-checkpoint, 2026-07-07)

- **First 2.4 slice:** SEO + share cards (over Source-Management UI / auto-discovery / onboarding).
- **Gating model:** *Landing + teaser feed.* Public landing page shows a small read-only
  teaser (latest headlines) as indexable content; **sign-in unlocks the full app**
  (all items, filters, feedback, saved views, per-user ranking, source mgmt).
- **Anon is fully NON-INTERACTIVE (refined 2026-07-07):** an anonymous visitor cannot click
  *anything* that acts on the app — no tabs, filters, feedback, navigation, or item links.
  The teaser is a static visual preview; the **only** actionable elements are **Sign in /
  Sign up**. All interactivity is unlocked by signing in.
- **Teaser size: 4–5 items** (use **5**) — just enough for a taste + indexable content.
- **Conversion copy:** the landing/sign-in surface explicitly states that to see the
  **top-rated / personalized** content they must **sign up or sign in**.
- Product/brand: **The AI Chronicles** · https://theaichronicles.ai (owner-run; owner = OWNER_EMAIL).

## Scope — IN

1. **Metadata foundation** — `metadataBase`, default title-template, description, OpenGraph +
   Twitter card defaults in the root layout.
2. **Public landing at `/`** — hero + value prop + a **read-only, NON-INTERACTIVE teaser**
   (latest **5** public headlines, sanitized, rendered as static text — item links inert for
   anon) + prominent **Sign in / Sign up** CTA with copy that says signing in unlocks the
   **top-rated / personalized** feed + `WebSite`/`Organization` JSON-LD. Server-rendered,
   fully indexable. No tabs/filters/feedback/nav are clickable for anon.
3. **Gated app moved to `/feed`** — the current feed/app (`app/page.tsx`) relocates to `/feed`,
   requiring a session. Signed-in users land on `/feed`; anon hitting `/feed` (or other app
   routes) is redirected to `/sign-in`. Signed-in users visiting `/` are sent to `/feed`.
4. **Dynamic OG share image** — `opengraph-image` (Next `ImageResponse`) for the landing so
   shared links render a branded card. No untrusted text rendered unsanitized.
5. **`sitemap.ts` + `robots.ts`** — sitemap lists public routes (landing); robots **allows**
   the landing, **disallows** `/feed`, `/settings`, `/sources`, `/sign-in`, `/api/*`, and those
   gated routes emit `noindex`.

## Scope — OUT (later 2.4 slices)

- In-app Source Management UI · automated source discovery + review digest · richer onboarding
  flows · per-item share images/pages · newsletter/email capture.

## Access & Security

- Teaser uses the **anon** (RLS) client; the RLS policy already allows anon read on
  `sources`/`items`. Count is capped **server-side** (~12); no personalization fields leak
  (anon already nulls legacy per-user columns per 2.2).
- Gated routes enforce the session **server-side** (redirect if unauthenticated) — not just UI.
- `robots` `noindex` on all gated routes + API; sitemap only lists public URLs.
- OG image: only render trusted/brand strings (no raw ingested content).
- No new secrets; no new external deps beyond Next built-ins (`ImageResponse`, metadata,
  sitemap/robots conventions).

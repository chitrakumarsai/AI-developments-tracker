# PRD — Subphase 1.2 Slice 4: Hugging Face connector

**Category:** Research Papers + LLM & Other Models · **Type:** `api` (new provider under existing host router) · Builds on the GitHub connector (1.2 slice 3).

## Problem
The catalog ingests arXiv + company blogs (RSS) and notable GitHub repos (API), but two of the highest-signal AI surfaces — **Hugging Face daily curated papers** and **trending models** — aren't tracked. Both are public JSON APIs and fit the link-first model exactly.

## Goal
Add one Hugging Face provider that feeds two catalog rows, with **zero new architecture** — it slots into the existing `api` host router next to GitHub.

## Scope (this slice)
- **HF Papers** (`/api/daily_papers`) → `Research Papers`, link to `huggingface.co/papers/{id}`.
- **HF Trending models** (`/api/models?sort=trendingScore&direction=-1&limit=50`) → `LLM & Other Models`, link to `huggingface.co/{id}`.
- Dispatch by URL **path** within one `huggingfaceConnector`.
- Token `HUGGINGFACE_TOKEN` (server-only, optional — endpoints are public; token raises rate limits).

## Out of scope
- HF datasets/Spaces (later, if wanted).
- Hacker News, Reddit, newsletters — subsequent 1.2 slices.
- Summaries / relevance scoring (1.4).

## Preferences (locked via preference-checkpoint)
| Decision | Choice |
|---|---|
| HF content | Daily Papers + Trending models |
| Trending sort | `sort=trendingScore` |
| Token env name | `HUGGINGFACE_TOKEN` |
| Delivery | Thin slice, one connector |

## Acceptance criteria
- [ ] `huggingface.co` routes to the HF connector via the api router.
- [ ] Papers map to Research Papers (link-first to HF paper page); models map to LLM & Other Models.
- [ ] All text sanitized; links via `sanitizeUrl`; SSRF guard on the fetch URL.
- [ ] Warns (never throws) on unsupported path / non-200 / fetch reject / missing token.
- [ ] Two `sources` rows added; new HF endpoint = catalog edit, no code.
- [ ] Tests green, ≥80% new-code coverage; full suite + typecheck clean.
- [ ] Live-verified; re-run dedupes to 0 on unique `url`.

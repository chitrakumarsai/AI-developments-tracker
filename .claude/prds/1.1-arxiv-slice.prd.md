# PRD — Subphase 1.1: First Vertical Slice (arXiv)

## Problem

The scaffold (1.0) boots an empty app wired to Supabase but shows no content. We need the **first end-to-end path** — one real source fetched, stored, and displayed on a mobile-first feed — proving the full pipeline and establishing the **reusable connector template** every later source (1.2) copies.

## Goal

Open the app on a phone viewport and see recent arXiv papers from seeded sources; tap a paper to open the original at arxiv.org. Re-running ingestion never duplicates items.

## Non-goals (explicitly deferred)

- Scheduled/automatic refresh → **1.3** (this slice triggers ingestion manually via a route).
- Thumbs up/down, ranking, LLM summaries/relevance → **1.4**.
- Filters, search, sort, saved views, read-state UI → **1.5**.
- Any source other than arXiv → **1.2**.
- Auth / multi-user / RLS → **Phase 2**.

## Recorded preferences (preference-checkpoint, 2026-06-23)

| Decision | Choice |
|---|---|
| **arXiv categories** | `cs.AI`, `cs.LG`, `cs.CL`, `cs.CV` — one seeded `sources` row (RSS feed) each |
| **Feed card layout** | **Editorial list** — title-forward, generous spacing, summary snippet, source/date/→ affordance (matches scaffolded Editorial/Swiss system) |
| **Abstract handling** | Store the **full abstract**; show it **expandable inline** (collapsed by default to a few lines, tap to expand) |
| **Volume** | **Latest 50** items per source per run |

## Users & scope

Single user (the owner), local-only, no auth (Phase 1).

## Success criteria

1. Running the ingest trigger populates `items` from real arXiv data across the 4 seeded feeds.
2. The feed renders those items on a 320–768px viewport; tapping opens the original at arxiv.org in a new tab (`rel="noopener noreferrer"`).
3. The full abstract is stored and expandable inline; collapsed by default.
4. Re-running ingest produces **zero** duplicate items (dedupe on `items.url`).
5. Connector, persistence, and feed API have unit/integration tests; one E2E passes; **≥80% coverage on new code**.
6. All ingested fields are **sanitized** before storage/render (untrusted input — §12.7); no `dangerouslySetInnerHTML`.

## Constraints & risks

- **arXiv feed format** — `http://export.arxiv.org/rss/<cat>`; abstracts may carry HTML entities/whitespace noise → normalize/sanitize. Mitigation: parse with `rss-parser`, sanitize text, test against a saved fixture (no live network in unit tests).
- **Untrusted content** — treat all feed fields as untrusted; render as text only.
- **Service-role key** — ingestion writes server-side only; never exposed to the browser (§21).
- **Dedupe correctness** — unique constraint on `items.url` + upsert (`ignoreDuplicates`); covered by a re-run test.

## Acceptance demo

`supabase start` → run migration + seed → `POST /api/ingest/run` (all active sources) → open `/` on a mobile viewport → see arXiv items, expand an abstract, tap through to arxiv.org → re-run ingest → count unchanged.
